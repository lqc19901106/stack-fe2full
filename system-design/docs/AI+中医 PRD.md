**产品范围**：
- 中医体质辨识与健康档案

## 二、产品概述

### 2.1 产品愿景

打造一款 **以 AI 赋能的中医健康产品**，将传统中医的望闻问切、辨证论治、体质学说与现代 AI 技术相结合，为用户提供体质辨识、症状分析、养生调理建议及中医师问诊服务，助力用户了解自身体质、科学调理、预防未病。

### 2.2 产品定位

| 维度 | 说明 |
|------|------|
| **目标用户** | 关注中医养生、亚健康调理、慢病管理的 C 端用户 |
| **核心价值** | 体质辨识 + AI 辨证参考 + 个性化养生 + 中医师问诊 |
| **差异化** | 中医理论驱动、AI 辅助四诊、体质个性化、轻问诊与深度问诊结合 |
| **使用场景** | 日常体质自测、亚健康调理、慢性病辅助管理、中医师咨询、养生知识学习 |

### 2.3 核心价值主张

1. **体质可识**：基于中医体质学说（九种体质等），通过问卷或舌/面图像，帮助用户了解自身体质类型及倾向。
2. **辨证有据**：AI 根据症状、舌象、体质等信息，给出辨证参考与调理方向，**仅供用户与医生参考，不替代医师诊断**。
3. **调理可循**：提供个性化食疗、起居、穴位、艾灸、情志等养生建议，贴合体质与节气。
4. **问诊可达**：连接执业中医师，支持图文/音视频问诊，处方由医师开具，合规可追溯。

### 2.4 竞争分析与差异化

| 竞品 | 主要特点 | 我们的差异 |
|------|----------|------------|
| **平安好医生 / 好大夫** | 大平台、中西医均有、品牌医生 | 中医垂直、体质+辨证+舌诊 AI 全链路、养生调理闭环 |
| **微医 / 丁香医生** | 预约挂号、健康科普 | AI 辨证参考、舌象分析、体质个性化养生 |
| **固生堂 / 同仁堂** | 线下中医馆、品牌背书 | 线上轻问诊+AI 自助体质/辨证，成本更低、触达更广 |
| **中医世家 / 问止中医** | 中医问诊、处方配送 | AI 辅助四诊、体质辨识前置、养生与问诊一体化 |
| **春雨医生 / 妙手医生** | 综合问诊、偏西医 | 中医理论驱动、九种体质、节气养生、穴位/食疗推荐 |

**核心竞争力**：中医理论 + AI 四诊辅助 + 体质个性化 + 养生与问诊闭环 + 合规可追溯

### 2.5 产品 KPI 与商业化

**用户侧 KPI**：DAU/MAU、体质辨识完成率、辨证参考使用率、养生内容阅读/收藏率、问诊转化率（发起→完成）、复诊率、NPS。

**医生侧 KPI**：活跃中医师数、人均接诊量、平均响应时长、处方合规率、患者评价与投诉率。

**商业化模式**：

| 模式 | 说明 | 优先级 |
|------|------|--------|
| **问诊服务费** | 按次或按时长向患者收费，与医生分成 | P0 |
| **会员/订阅** | 月/年费会员：免费体质复测、养生内容、问诊折扣 | P1 |
| **处方流转** | 处方对接合作药房，药品差价或配送费 | P1 |
| **B2B/企业健康** | 企业员工中医养生服务、保险合作 | P2 |
| **增值服务** | 舌象深度解读、调理计划定制、专家号预约 | P2 |

## 三、中医理论基础简述

### 3.1 核心概念（产品需体现）

| 概念 | 说明 | 产品应用 |
|------|------|----------|
| **望闻问切** | 四诊合参：望（面色、舌象、形态）、闻（声音、气味）、问（症状、病史）、切（脉象） | 问诊表单、舌/面图像上传、症状结构化 |
| **辨证论治** | 根据证候推断病机，再施治 | AI 证型推断参考、方剂/穴位推荐方向 |
| **体质学说** | 九种体质（平和、气虚、阳虚、阴虚、痰湿、湿热、血瘀、气郁、特禀） | 体质辨识问卷、体质画像与调理建议 |
| **经络穴位** | 十二经络、常用穴位 | 穴位推荐、艾灸/按摩指导 |
| **整体观** | 天人合一、脏腑相关 | 节气养生、情志与体质关联 |
| **治未病** | 未病先防、既病防变 | 体质调理、亚健康干预、提醒与计划 |

### 3.2 合规与安全原则

- **AI 输出定位**：所有 AI 给出的辨证、方剂、穴位等建议，均标注「仅供参考，不构成诊断与治疗」，不可替代执业医师判断。
- **处方权**：中药处方仅可由平台入驻的执业中医师开具，AI 仅作辅助参考。
- **风险提示**：涉及孕妇、儿童、急症等情况，明确引导线下就医。

## 四、目标用户与使用场景

### 4.1 目标用户画像

| 用户类型 | 典型特征 | 核心诉求 | 典型目标 |
|----------|----------|----------|----------|
| **养生爱好者** | 认同中医理念，注重日常调理 | 了解体质、获取养生建议 | 体质改善、少生病 |
| **亚健康人群** | 疲乏、失眠、易感冒等，体检无大病 | 找到调理方向、改善状态 | 精力提升、睡眠改善 |
| **慢病调理** | 高血压、糖尿病等需长期调理 | 中医辅助调理、与西医协同 | 症状稳定、减少用药 |
| **女性健康** | 经期、备孕、更年期等 | 调经、养颜、舒压 | 经期规律、气色改善 |
| **家长** | 孩子易感冒、消化不良等 | 小儿推拿、食疗建议 | 增强体质、少生病 |
| **中医师** | 执业中医，入驻平台接诊 | 提高接诊效率、患者管理 | 接诊量、复诊率 |

### 4.2 典型使用场景

| 场景 | 描述 | 关键需求 |
|------|------|----------|
| **初次体验** | 新用户想了解自身体质 | 体质问卷或舌象上传，快速得出体质类型 |
| **日常调理** | 根据体质与节气获取养生建议 | 食疗、起居、穴位、情志建议，可收藏 |
| **不适自查** | 轻微不适（如头痛、失眠）想先了解 | 症状描述 → AI 辨证参考 → 建议就医或调理 |
| **舌象自测** | 拍照上传舌象，获取舌诊参考 | 舌质、舌苔、裂纹等分析，体质/证型关联 |
| **预约中医师** | 需要专业辨证与处方 | 选医生、选时段、图文/音视频问诊 |
| **复诊续方** | 慢病需定期调方 | 关联历史病历、快捷续方、提醒复诊 |
| **节气养生** | 立春、夏至、秋分等节点 | 节气养生内容、个性化提醒 |

### 4.3 核心用户旅程

**旅程一：体质自测与养生入门**

```
注册/登录 → 体质辨识问卷(5~10min) → 体质报告(主体质+兼夹+调理概要) → 个性化养生推荐(食疗/穴位) → 收藏/制定调理计划
```

**旅程二：症状自查与 AI 辨证**

```
输入症状(选择/自由文本) → 可选上传舌象 → AI 辨证参考(证型+调理方向) → 风险判断(急症/孕妇→引导就医) → 推荐问诊/养生建议
```

**旅程三：中医师问诊**

```
选医生(科室/擅长/排班) → 填写主诉+携带体质/舌象(若有) → 支付 → 图文/音视频问诊 → 医生辨证+处方 → 处方审核(可选) → 取药(配送/到店)
```

**旅程四：复诊续方与调理跟进**

```
复诊提醒推送 → 关联历史病历/处方 → 续方/调方 → 调理打卡(可选) → 复测体质(周期)
```

├── 中医师问诊模块
├── 健康档案与计划模块

### 5.2 体质辨识模块

| 需求 ID | 需求描述 | 优先级 | 验收标准 |
|---------|----------|--------|----------|
| T-01 | 支持中医体质辨识问卷（基于《中医体质分类与判定》等标准） | P0 | 问卷覆盖九种体质，题目数量合理（如 60~80 题） |
| T-02 | 问卷完成后自动计算各体质得分，给出主要体质类型及倾向 | P0 | 展示体质分布图/雷达图，主体质 + 兼夹体质 |
| T-03 | 体质结果可结合舌象（若用户已上传）进行综合解读 | P1 | 舌象与体质结论可交叉参考 |
| T-04 | 体质报告包含体质特点、易患倾向、调理原则概要 | P0 | 每类体质有标准解读模板 |
| T-05 | 支持体质复测，记录历史体质变化趋势 | P1 | 可查看多次测评结果对比 |
| T-06 | 体质结果作为后续养生建议、辨证参考的输入 | P0 | 与养生、问诊模块数据贯通 |

### 5.3 症状问诊与辨证参考模块

| 需求 ID | 需求描述 | 优先级 | 验收标准 |
|---------|----------|--------|----------|
| S-01 | 用户可自由描述或选择症状（头痛、失眠、乏力等） | P0 | 支持结构化选择 + 自由文本补充 |
| S-02 | 症状库按中医证候分类，支持多选与程度标注 | P0 | 常见症状覆盖 100+，支持轻/中/重 |
| S-03 | AI 根据症状、体质、舌象（若有）给出辨证参考 | P0 | 输出可能证型、病机简述、调理方向（明确标注仅供参考） |
| S-04 | 辨证参考可关联推荐调理方式（食疗、穴位、中成药方向等） | P0 | 每类证型有对应建议，注明「不替代医生」 |
| S-05 | 对急症、孕产妇、儿童等场景给出风险提示并引导就医 | P0 | 关键词触发提示，不可跳过 |
| S-06 | 支持「推荐问诊」：建议用户预约中医师进一步辨证 | P1 | 可一键跳转预约页，并携带症状摘要 |

### 5.4 舌诊/面诊图像分析模块

| 需求 ID | 需求描述 | 优先级 | 验收标准 |
|---------|----------|--------|----------|
| L-01 | 用户可上传舌象照片，AI 辅助分析舌质、舌苔、裂纹等 | P0 | 输出舌质颜色、舌苔厚薄/颜色、齿痕、裂纹等描述 |

| L-05 | 舌象历史可保存，支持前后对比 | P1 | 可查看历史舌象及解读 |
| L-06 | 明确标注「AI 辅助分析，仅供参考，不构成诊断」 | P0 | 分析结果页有醒目提示 |

### 5.10 中医师端功能

## 八、验收与优先级

### 8.1 优先级定义

| 优先级 | 说明 | 发布要求 |
|--------|------|----------|
| **P0** | 核心功能，MVP 必须有 | 首版必须实现 |
| **P1** | 重要功能，影响体验与合规 | 首版或第二版 |
| **P2** | 增强功能，可后续迭代 | 按资源排期 |

### 8.2 MVP 建议范围

**建议 MVP 包含**：
- 体质辨识问卷与体质报告
- 症状采集与 AI 辨证参考（含风险提示）
- 基础养生建议（食疗、穴位，按体质推荐）
- 中医师图文问诊与处方
- 用户注册登录、实名认证、健康档案
- 舌象上传与 AI 分析（若资源允许）

**MVP 成功标准**：
- 体质辨识完成率 ≥ 60%
- 完成至少 1 单问诊（发起→支付→完成）
- 医生端可查看患者体质与 AI 辨证结果
- 无重大合规与安全事件

**可延后**：
- 面诊图像分析
- 视频问诊
- 节气养生推送
- 调理计划与打卡
- AI 问答
- 药师审核流程

### 8.3 实施路线与里程碑

| 阶段 | 时间（示例） | 交付物 | 关键目标 |
|------|--------------|--------|----------|
| **M0 筹备** | 0~4 周 | 需求冻结、技术选型、合规与资质梳理 | 明确资质要求、知识库初版 |
| **M1 MVP** | 4~12 周 | 体质辨识、症状采集、AI 辨证、养生建议、图文问诊、医生入驻 | 内测上线、完成首单问诊 |
| **M2 增强** | 12~20 周 | 舌象 AI 分析、复诊续方、会员体系、药师审核（可选）、数据报表 | 公测、DAU 破千 |
| **M3 扩展** | 20~28 周 | 节气养生、调理计划、AI 问答、视频问诊、B2B 试点 | 商业化验证、复诊率达标 |
| **M4 优化** | 28 周+ | 面诊、调理打卡、企业健康、处方流转深化 | 规模化运营、盈利模型跑通 |

## 九、约束与假设

### 9.1 约束

- 用户身份与登录可复用现有账户体系
- 消息推送可复用《公用-订阅、消息推送系统设计》
- 问诊与处方流程需符合互联网诊疗相关法规
- 中医师需具备执业资格，平台需具备相应资质

### 9.4 依赖与对接

| 依赖类型 | 说明 | 备注 |
|----------|------|------|
| **LLM 服务** | AI 辨证、病历草稿、AI 问答 | 需中医领域微调或 prompt 优化 |
| **舌诊/图像模型** | 舌象、面诊图像分析 | 可自研或对接第三方 API |
| **支付** | 微信支付、支付宝 | 问诊费、会员费、处方费 |
| **药房/配送** | 处方流转、药品配送 | 可选，MVP 后可对接 |
| **实名认证** | 身份证、人脸核身 | 问诊前必须完成 |

### 9.2 假设

- 用户对中医有基本认知或愿意了解
- 舌象等图像在常规光线下可满足分析需求（不要求专业设备）
- AI 辨证在多数常见证型下可给出有价值参考
- 用户接受「AI 辅助、医师决策」的产品定位

### 9.3 风险与应对

| 风险类型 | 风险描述 | 应对措施 |
|----------|----------|----------|
| **合规风险** | 互联网中医诊疗资质、处方流转监管 | 提前梳理《互联网诊疗监管细则》等，确保平台与医生资质齐备；处方审核流程可追溯 |
| **医疗风险** | AI 辨证误判、用户延误就医 | AI 输出统一标注「仅供参考」；急症/孕妇/儿童强制引导就医；症状库设风险关键词触发 |
| **技术风险** | LLM/舌诊模型不可用或延迟 | 设计降级策略：AI 不可用时提示人工入口；舌诊可跳过，仅用症状+体质辨证 |
| **数据风险** | 健康数据泄露、滥用 | 数据分类分级、加密存储、访问审计；用户可导出/删除数据 |
| **运营风险** | 中医师供给不足、质量参差 | 分阶段拓展医生资源；入驻审核与评价体系；质控抽检 |
| **产品风险** | 体质/辨证 AI 准确度不足影响信任 | 建立专家抽检机制；持续优化 prompt 与模型；用户反馈闭环 |
