# ES6 面试问答版（背诵速记）

这版适合面试前快速过题，结构为：**面试官问什么 -> 你怎么答 -> 易错点**。

## 1) `var`、`let`、`const` 有什么区别？

**标准回答：**
- `var` 是函数作用域，存在变量提升，可重复声明。
- `let`、`const` 是块级作用域，存在暂时性死区（TDZ），不能重复声明。
- `const` 不能重新赋值，但若值是对象，可修改对象内部属性。

**易错点：**
- 说 `const` 对象“完全不可变”是错的，它只是“绑定不可变”。

---

## 2) 什么是暂时性死区（TDZ）？

**标准回答：**
- 在 `let/const` 声明前访问变量，会抛出 `ReferenceError`。
- 从块级作用域开始到声明语句执行前，这段区域叫 TDZ。

**示例：**

```js
{
  // console.log(a); // ReferenceError
  let a = 1;
}
```

**易错点：**
- 不是“变量不存在”，而是“存在但不可访问”。

---

## 3) 为什么 `let` 能解决 `for + setTimeout` 问题？

**标准回答：**
- `let` 在每次循环会形成新的块级绑定，每次回调闭包拿到的是当前轮次独立的 `i`。
- `var` 只有一个共享绑定，回调执行时循环通常已结束。

---

## 4) 箭头函数和普通函数的 `this` 区别？

**标准回答：**
- 普通函数的 `this` 由调用方式决定（谁调用指向谁，或 `call/apply/bind` 指定）。
- 箭头函数没有自己的 `this`，继承外层词法作用域的 `this`。

**易错点：**
- 不能把箭头函数当构造函数用（不能 `new`）。

---

## 5) 箭头函数为什么没有 `arguments`？

**标准回答：**
- 箭头函数没有自己的 `arguments`，需要用剩余参数 `...args` 获取参数列表。

```js
const fn = (...args) => args.length;
```

---

## 6) 解构赋值默认值什么时候生效？

**标准回答：**
- 仅在属性值严格等于 `undefined` 时触发默认值。
- 若值是 `null`，默认值不会生效。

```js
const { x = 1 } = { x: undefined }; // 1
const { y = 1 } = { y: null }; // null
```

---

## 7) 扩展运算符是深拷贝吗？

**标准回答：**
- 不是，展开运算符和 `Object.assign` 都是浅拷贝。
- 嵌套对象仍共享引用。

**易错点：**
- 面试中常被追问“如何深拷贝”，可答 `structuredClone`、递归、自定义方案或库函数。

---

## 8) Promise 有哪几种状态？状态能否逆转？

**标准回答：**
- 三种状态：`pending`、`fulfilled`、`rejected`。
- 只能从 `pending` 变成 `fulfilled/rejected`，一旦改变不可逆。

---

## 9) `Promise.all`、`allSettled`、`race`、`any` 区别？

**标准回答：**
- `Promise.all`：全部成功才成功；任一失败则整体失败。
- `Promise.allSettled`：等待全部结束，返回每个任务的状态，不会因单个失败中断。
- `Promise.race`：谁先结束（成功或失败）就以谁为结果。
- `Promise.any`：任一成功即成功；全部失败才失败（`AggregateError`）。

---

## 10) `async/await` 本质是什么？

**标准回答：**
- `async` 函数本质返回 Promise。
- `await` 是 Promise 链式调用的语法糖，让异步代码更像同步代码。

**易错点：**
- `await` 只能在 `async` 函数内（或顶层模块）使用。

---

## 11) `async` 函数里 `return` 和 `throw` 分别等价什么？

**标准回答：**
- `return value` 等价于 `Promise.resolve(value)`。
- `throw err` 等价于 `Promise.reject(err)`。

---

## 12) 如何优雅处理多个异步请求？

**标准回答：**
- 无依赖并行：`Promise.all`。
- 允许部分失败：`Promise.allSettled`。
- 顺序依赖：按 `await` 串行执行。
- 高并发控制：配合任务队列/并发池。

---

## 13) ES6 的 `class` 是不是新的面向对象机制？

**标准回答：**
- 不是全新机制，`class` 是基于原型链的语法糖。
- 实例方法仍定义在 `prototype` 上。

---

## 14) `extends` 和 `super` 的作用？

**标准回答：**
- `extends` 建立子类对父类的继承关系。
- 子类构造函数中必须先调用 `super()` 才能使用 `this`。

**易错点：**
- 在子类构造函数里先用 `this` 再 `super()` 会报错。

---

## 15) `export default` 和具名导出区别？

**标准回答：**
- `export default` 每个模块只能一个，导入时名称可自定义。
- 具名导出可多个，导入时通常按名称匹配（可 `as` 重命名）。

```js
import anyName, { foo as bar } from "./mod.js";
```

---

## 16) 为什么说 ES Module 是“静态依赖”？

**标准回答：**
- `import/export` 在编译阶段就能确定依赖关系，有利于 Tree Shaking 和静态分析。
- 对比 CommonJS 的 `require`，后者可运行时动态加载。

---

## 17) `Map` 和普通对象 `{}` 区别？

**标准回答：**
- `Map` 的 key 可为任意类型；对象 key 会被转成字符串（或 Symbol）。
- `Map` 更适合频繁增删查映射关系，语义更清晰，且有 `size`。

---

## 18) `Set` 典型应用场景？

**标准回答：**
- 数组去重、成员存在性判断、交并差运算。

```js
const unique = [...new Set(arr)];
```

---

## 19) `Symbol` 有什么实际价值？

**标准回答：**
- 提供唯一键，避免对象属性名冲突。
- 可用于定义对象内部“半私有”属性或扩展协议（如 `Symbol.iterator`）。

---

## 20) `for...in` 和 `for...of` 区别？

**标准回答：**
- `for...in` 遍历可枚举属性名（含原型链可枚举属性，不推荐遍历数组）。
- `for...of` 遍历可迭代对象的值（数组、字符串、Map、Set 等）。

---

## 21) 什么是 Iterator？

**标准回答：**
- 迭代器是一个对象，调用 `next()` 返回 `{ value, done }`。
- 实现了 `Symbol.iterator` 的对象是可迭代对象，可被 `for...of` 消费。

---

## 22) Generator 有什么用？

**标准回答：**
- `Generator` 允许函数暂停与恢复执行，按需产出值。
- 可用于自定义迭代流程、状态机、惰性序列。

---

## 23) `Proxy` 和 `defineProperty` 比有什么优势？

**标准回答：**
- `Proxy` 拦截能力更全面（如 `get/set/has/deleteProperty/apply/construct`）。
- 可直接代理整个对象，不用逐个属性劫持。

---

## 24) `Reflect` 为什么常和 `Proxy` 一起用？

**标准回答：**
- `Reflect` 提供与 Proxy trap 对应的默认行为，便于在增强逻辑后回退到标准操作。
- 代码更规范，也便于保持行为一致性。

---

## 25) 什么是尾调用优化（TCO）？ES6 支持了吗？

**标准回答：**
- 规范提出过“严格模式下的尾调用优化”，但主流引擎普遍未完整实现。
- 面试可回答“理论有、实际兼容有限，工程中不依赖”。

---

## 26) `Object.is` 和 `===` 区别？

**标准回答：**
- `Object.is(NaN, NaN)` 为 `true`，而 `NaN === NaN` 为 `false`。
- `Object.is(+0, -0)` 为 `false`，而 `+0 === -0` 为 `true`。

---

## 27) 常见事件循环追问怎么答？

**标准回答（简版）：**
- 同步任务先执行。
- 微任务（如 Promise 回调）在当前宏任务结束后立即清空。
- 然后进入下一轮宏任务（如 `setTimeout`）。

**易错点：**
- 说 “`setTimeout(fn, 0)` 一定立即执行” 是错的，它只是尽快进入宏任务队列。

---

## 28) 面试时如何判断该用微调语言回答还是深入原理？

**建议答法：**
- 第一轮先给结论 + 关键词（30 秒内）。
- 面试官追问时再补底层（原型链、微任务队列、静态分析等）。
- 带一个短代码例子最加分。

---

## 29) ES6 高频易错点总览

- 把 `const` 理解成“对象不可变”。
- 忘记 `await` 需要 `try...catch` 做错误处理。
- 把 `for...in` 用在数组上。
- 误以为展开运算符是深拷贝。
- 把 `class` 当成与原型链无关的新机制。
- 不清楚 `all` 与 `allSettled` 的场景差异。
