# 全栈工程师需要的 Skills

全栈工程师不是简单地“前端 + 后端”都会一点，而是能够从需求理解、页面实现、服务开发、数据设计、部署上线、监控运维到问题排查，完整交付一个可运行、可维护的软件系统。

## Skills 目录
- [Code Review Skill](./code-review.md)

## 1. 计算机基础
- **数据结构与算法**: 数组、链表、栈、队列、树、图、哈希表、排序、搜索、动态规划。
- **计算机网络**: HTTP/HTTPS、TCP/IP、DNS、WebSocket、CDN、代理、缓存、跨域。
- **操作系统**: 进程、线程、文件系统、内存、I/O、Linux 常用命令。
- **数据库基础**: 事务、索引、锁、隔离级别、执行计划、范式与反范式。
- **软件工程**: 模块化、分层设计、设计模式、代码规范、测试、版本控制。

## 2. 前端开发能力
- **HTML/CSS/JavaScript**: 语义化、布局、响应式、事件模型、异步编程、浏览器渲染机制。
- **TypeScript**: 类型建模、泛型、类型收窄、工具类型、类型安全的接口定义。
- **前端框架**: React / Vue / Svelte，理解组件、状态、生命周期、Hooks、路由。
- **工程化**: Vite、Webpack、ESLint、Prettier、pnpm/npm、模块化、构建优化。
- **UI 与交互**: 表单、表格、弹窗、上传、图表、空状态、加载状态、错误状态。
- **状态管理**: Zustand、Redux、Pinia、React Query / TanStack Query。
- **性能优化**: 首屏加载、懒加载、代码分割、缓存、虚拟列表、减少重渲染。
- **可访问性**: 语义标签、键盘操作、ARIA、颜色对比度、表单 label。

## 3. 后端开发能力
- **服务框架**: Node.js/NestJS/Express、Java Spring Boot、Go Gin、Rust Axum/Actix 等。
- **API 设计**: REST、GraphQL、gRPC、WebSocket、统一响应、错误码、分页、排序、过滤。
- **业务建模**: DTO、领域模型、状态机、权限模型、幂等、事务、并发控制。
- **认证授权**: Session、JWT、OAuth2、RBAC、ABAC、多租户隔离。
- **异步任务**: 定时任务、后台任务、消息队列、重试、死信队列、任务状态追踪。
- **文件处理**: 上传、下载、预览、分片、对象存储、类型校验、权限控制。
- **错误处理**: 参数错误、业务错误、系统错误、第三方错误、统一异常处理。
- **服务质量**: 超时、重试、熔断、限流、降级、健康检查。

## 4. 数据库与数据能力
- **关系型数据库**: MySQL、PostgreSQL、SQLite，表设计、索引、事务、慢查询优化。
- **NoSQL**: Redis、MongoDB、Elasticsearch，根据场景选择合适存储。
- **缓存设计**: 缓存 key、TTL、缓存穿透、击穿、雪崩、一致性策略。
- **数据建模**: 实体关系、字段类型、唯一约束、软删除、审计字段、数据生命周期。
- **数据迁移**: migration、回滚、兼容变更、数据回填、双写和校验。
- **搜索与分析**: 全文检索、聚合查询、日志分析、指标统计。

## 5. DevOps 与部署能力
- **Git 工作流**: branch、commit、merge、rebase、tag、PR/MR、代码评审。
- **CI/CD**: 自动构建、测试、扫描、打包、部署、回滚。
- **容器化**: Dockerfile、Docker Compose、多阶段构建、镜像优化。
- **Kubernetes 基础**: Deployment、Service、Ingress、ConfigMap、Secret、HPA、探针。
- **环境管理**: dev/test/staging/prod 环境隔离，配置和密钥管理。
- **日志与监控**: 应用日志、系统指标、业务指标、链路追踪、告警。
- **部署策略**: 滚动发布、蓝绿发布、灰度发布、金丝雀发布、特性开关。
- **故障处理**: 定位、止血、回滚、恢复、复盘。

## 6. 安全能力
- **Web 安全**: XSS、CSRF、点击劫持、CORS、CSP、Cookie 安全。
- **API 安全**: 鉴权、权限、限流、参数校验、防重放、错误信息控制。
- **数据安全**: 加密、脱敏、访问控制、审计、备份、隐私保护。
- **依赖安全**: 依赖漏洞扫描、镜像扫描、供应链安全、版本锁定。
- **密钥管理**: 环境变量、Secret、密钥轮换，避免密钥进入仓库和日志。
- **文件安全**: 类型白名单、大小限制、病毒扫描、私有存储、下载鉴权。

## 7. 架构与系统设计能力
- **分层架构**: Controller、Service、Repository、DTO、Domain Model。
- **服务拆分**: 单体、模块化单体、微服务、服务边界、数据 owner。
- **系统集成**: API 网关、服务发现、消息队列、第三方接口、Webhook。
- **高可用设计**: 冗余、故障隔离、限流、降级、熔断、备份恢复。
- **性能设计**: 缓存、异步化、批处理、连接池、分页、索引、容量规划。
- **可观测性设计**: traceId、结构化日志、指标、链路追踪、SLO。
- **架构文档**: 系统上下文图、服务关系图、数据流图、部署图、ADR。

## 8. 测试与质量保障
- **单元测试**: 工具函数、业务逻辑、组件、Hook、Service。
- **集成测试**: API、数据库、缓存、消息队列、第三方 mock。
- **端到端测试**: Playwright、Cypress，覆盖关键用户路径。
- **接口测试**: 请求参数、响应结构、错误码、权限、边界条件。
- **性能测试**: 压测、响应时间、吞吐量、资源占用。
- **质量门禁**: lint、type check、test coverage、代码扫描、CI 阻断。

## 9. 产品与业务理解
- **需求拆解**: 从业务目标拆成页面、接口、数据、流程和验收标准。
- **用户体验**: 加载、空状态、错误提示、确认操作、撤销、表单校验。
- **业务流程**: 状态流转、权限规则、审批流程、异常流程。
- **数据指标**: 埋点、转化率、留存、活跃、错误率、业务漏斗。
- **沟通协作**: 和产品、设计、后端、测试、运维、安全协同。

## 10. AI 与工具化能力
- **AI 辅助编码**: 使用 Cursor、GitHub Copilot 等工具提升开发效率。
- **Prompt 能力**: 能清晰描述需求、约束、输入输出和验收标准。
- **AI 应用开发**: LLM API、RAG、Embedding、Agent、工具调用、流式输出。
- **本地模型应用**: llama.cpp、Ollama、ONNX Runtime、Candle、本地推理服务。
- **自动化工具**: 脚本、CLI、代码生成、文档生成、测试数据生成。
- **工程提效**: 自动生成接口类型、数据库类型、SDK、Mock、文档。

## 11. 常用工具
- **开发工具**: VS Code / Cursor、WebStorm、Chrome DevTools、Postman、Insomnia。
- **版本管理**: Git、GitHub、GitLab。
- **包管理**: npm、pnpm、yarn、Cargo、Maven、Gradle、Go Modules。
- **数据库工具**: DBeaver、DataGrip、TablePlus、RedisInsight。
- **接口与文档**: Swagger/OpenAPI、Apifox、Postman、Markdown、Mermaid。
- **部署运维**: Docker、Docker Compose、Kubernetes、Nginx、GitHub Actions。
- **监控日志**: Grafana、Prometheus、Loki、ELK、OpenTelemetry、Sentry。

## 12. 能力分级建议

### 初级全栈工程师
- 能独立完成简单页面和基础 CRUD 接口。
- 能理解数据库表结构和基础 SQL。
- 能使用 Git、npm、Docker Compose 跑起项目。
- 能根据文档调用接口并处理基础错误状态。

### 中级全栈工程师
- 能独立交付一个完整业务模块。
- 能设计接口、表结构、权限和基本异常处理。
- 能处理常见性能问题、部署问题和线上 bug。
- 能编写基础测试、文档和 CI 配置。

### 高级全栈工程师
- 能从需求到上线完整负责一个系统。
- 能设计可维护的前后端架构和服务边界。
- 能处理高并发、安全、可观测性、部署和回滚问题。
- 能推动工程规范、代码质量、自动化和团队效率提升。

### 资深全栈/技术负责人
- 能做系统架构、技术选型和长期演进规划。
- 能识别业务风险、架构风险、安全风险和交付风险。
- 能建立团队工程标准、DevOps 流程和质量体系。
- 能把 AI、自动化和平台能力融入研发流程。

## 13. 学习路线建议
1. 先掌握 HTML、CSS、JavaScript、TypeScript 和一个前端框架。
2. 学会一个后端技术栈，能写 API、连数据库、做鉴权和错误处理。
3. 掌握 SQL、索引、事务、缓存和基础数据建模。
4. 学会 Docker、CI/CD、日志、监控和基础部署。
5. 补齐安全、测试、性能优化和系统设计能力。
6. 逐步学习 AI 应用开发和自动化工具，提升工程效率。

## 14. 项目实践建议
- **个人博客系统**: 登录、文章管理、评论、搜索、部署。
- **任务管理系统**: 用户、项目、任务、状态流转、权限、通知。
- **电商后台**: 商品、订单、库存、支付回调、数据看板。
- **知识库系统**: 文档上传、全文检索、权限、版本管理。
- **AI 文档助手**: 文件解析、向量检索、本地模型问答、流式输出。
- **DevOps Demo**: Docker 化、CI/CD、监控、日志、灰度发布和回滚。

## 15. 核心判断标准
- 能不能把需求拆成可交付任务。
- 能不能写出可维护、可测试、可部署的代码。
- 能不能理解前端、后端、数据库和部署之间的关系。
- 能不能在出问题时快速定位是页面、接口、数据、配置还是环境问题。
- 能不能让系统不只是“能跑”，而是“可靠、安全、可维护、可演进”。
